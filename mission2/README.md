# Mission 2: Git과 함께하는 Python 첫 발자국 (퀴즈 게임)

## 1. 프로젝트 개요
Python 기본 문법과 객체 지향 프로그래밍(클래스)을 활용하여 터미널에서 동작하는 퀴즈 게임을 구현하였습니다. Git을 통한 버전 관리 및 협업(브랜치, 병합) 과정을 포함합니다.

## 2. 퀴즈 주제 선정 이유
**주제: Python 프로그래밍 기초**
학습자들이 Python의 기본적인 문법과 특징을 다시 한번 복습할 수 있도록 입문자 수준의 문제를 선정하였습니다.

## 3. 실행 방법
```bash
python mission2/main.py
```

## 4. 기능 목록
- **퀴즈 풀기**: 저장된 퀴즈를 순서대로 풀고 결과를 확인합니다.
- **퀴즈 추가**: 새로운 질문, 선택지, 정답을 입력하여 퀴즈를 등록합니다.
- **퀴즈 목록**: 현재 등록된 모든 퀴즈의 제목을 확인합니다.
- **점수 확인**: 최고 점수를 확인하고 갱신합니다.
- **데이터 저장**: `state.json`을 통해 퀴즈와 점수를 영구 저장합니다.

## 5. 파일 구조
- `main.py`: 퀴즈 게임의 핵심 로직 (Quiz, QuizGame 클래스)
- `state.json`: 퀴즈 데이터 및 최고 점수 저장 파일
- `README.md`: 프로젝트 문서

## 6. 데이터 파일 설명 (state.json)
- **quizzes**: 퀴즈 객체 배열 (question, choices, answer 필드)
- **best_score**: 현재까지 달성한 최고 점수 (숫자)

## 7. Git Workflow 실습 로그
- **Commits**: 10개 이상의 의미 있는 커밋 (Feat, Fix, Refactor, Docs 등)
- **Branch**: `feature/quiz-play` 브랜치 생성 및 `main` 병합 완료
- **Clone/Pull**: 로컬 복제본을 통한 기능 변경 및 동기화 테스트 수행

---
**학습 포인트**: 클래스를 통한 역할 분리(데이터 담당 vs 게임 관리 담당)의 이점을 이해하고, Git을 통해 개발 이력을 체계적으로 관리하는 습관을 기름.
