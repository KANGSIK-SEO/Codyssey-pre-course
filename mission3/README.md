# Mission 3: Mini NPU Simulator

## 1. 프로젝트 개요
AI 연산의 핵심인 MAC(Multiply-Accumulate) 연산을 시뮬레이션하고, 다양한 크기의 필터와 패턴을 비교하여 유사도를 판정하는 프로그램을 개발하였습니다. 또한 패턴 크기 증가에 따른 성능 변화를 분석합니다.

## 2. 주요 기능
- **MAC 연산**: 가중치(필터)와 입력(패턴)의 위치별 곱셈 및 누적 합 계산
- **라벨 정규화**: '+', 'x', 'cross' 등 다양한 입력을 'Cross', 'X'로 표준화
- **정밀도 처리**: 부동소수점 오차를 고려한 epsilon(1e-9) 기반 비교 정책
- **성능 분석**: 3x3부터 25x25까지 크기별 평균 연산 시간 측정

## 3. 실행 방법
\`\`\`bash
python mission3/main.py
\`\`\`
1번(사용자 입력) 또는 2번(data.json 분석) 모드를 선택하여 실행합니다.

### [실패 원인 및 원리 분석]
- **성공 요인 (0 FAIL)**: 본 시뮬레이터가 모든 케이스를 통과(PASS)한 이유는 **라벨 정규화**와 **허용 오차(epsilon) 비교 정책** 덕분입니다. 단순 문자열 비교나 엄격한 부동소수점 비교 대신, `+`를 `Cross`로 변환하고 `abs(score_a - score_b) < 1e-9`를 적용함으로써 데이터 형식의 다양성과 수치적 오차 문제를 해결했습니다.
- **시간 복잡도 분석**: MAC 연산은 $N \in \mathbb{R}^{n \times n}$ 행렬에 대해 중첩 반복문(nested loops)을 수행하므로 복잡도는 **$O(N^2)$**입니다. 실제 측정 결과에서도 데이터 크기 $N$이 5에서 25로 5배 증가할 때, 연산 상수는 약 25배($5^2$)에 가깝게 증가하는 $O(N^2)$의 비선형적 증가 추세를 확인할 수 있었습니다.
- **NPU의 필요성**: 현재는 단일 연산이지만, 실제 AI 모델처럼 수천 개의 필터를 처리할 경우 CPU의 직렬 연산($O(N^2)$ 반복)은 병목이 됩니다. 이를 병렬로 처리하는 NPU의 구조적 이점이 대규모 MAC 연산에서 왜 필수적인지 이론적으로 이해할 수 있었습니다.
